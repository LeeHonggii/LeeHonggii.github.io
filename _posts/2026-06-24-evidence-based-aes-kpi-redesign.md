# 도입 (왜 이 글)

자동 작문 채점(AES, Automated Essay Scoring) 시스템을 만들고 나서 가장 먼저 맞닥뜨린 질문은 "이게 얼마나 잘 맞아?"였다.

처음엔 단순하게 생각했다. Accuracy(정확도) 하나면 충분하지 않을까? AI 추천 점수가 실제 점수와 얼마나 일치하는지 비율로 보면 되는 거 아닌가?

결론부터 말하면, **그렇게 하면 심사장에서 무너진다.** 더 정확히는 — 실제로 잘 동작하는 시스템도 지표 선정이 잘못되면 방어가 안 된다. 이 글은 AES 평가 지표를 처음부터 다시 설계한 과정을 기록한다.

## Accuracy가 왜 문제인가

5점 척도로 글을 채점하는 시스템을 예로 들자. 실제 점수가 3점인데 AI가 2점을 줬다면 Accuracy 기준으로는 오답이다. 그런데 "1점 차이"와 "4점 차이"를 같은 오답으로 취급하는 건 직관적으로 이상하다.

더 근본적인 문제도 있다. **Accuracy는 분포를 숨긴다.** 모델이 운 좋게 3점과 4점만 찍어도 특정 데이터셋에서 높은 Accuracy가 나올 수 있다. 실제로 극단값(1점, 5점)에서 얼마나 틀리는지는 보이지 않는다.

국제 표준 기관들이 AES 평가에서 Accuracy를 단독 지표로 거의 쓰지 않는 이유가 여기 있다.

## AES 표준 지표 5종 설계

지표를 다시 짤 때 기준으로 삼은 건 두 가지였다: **실제로 쓰이는 학술 기준**과 **외부 심사자가 납득할 수 있는 근거**.

아래 5개 지표를 조합했다.

### 1. QWK (Quadratic Weighted Kappa, 이차가중카파)

채점자 간 일치도를 측정하는 가장 표준적인 지표다. AES 분야 논문에서 사실상 기본 지표로 사용된다. 단순 일치율과 달리 **점수 차이의 크기에 가중치를 준다** — 1점 차이보다 3점 차이를 더 나쁘게 본다.

```
QWK >= 0.70  # 실용 수준 최소 기준
             # 0.80 이상이면 높은 일치도로 인정
```

Landis & Koch(1977) 기준으로 0.61~0.80은 "상당한 일치(substantial agreement)"에 해당한다.

### 2. Pearson 상관계수 (Pearson Correlation Coefficient)

AI 점수와 기준 점수 사이의 선형 상관성을 측정한다. Spearman(스피어만) 상관계수도 일반적으로 쓸 수 있지만, AES 표준 문헌에서 근거를 제시하려면 **Pearson이 더 방어 가능**하다. TOEFL·GRE 채점 연구들이 주로 Pearson을 사용하기 때문이다.

```
Pearson >= 0.70
```

### 3. SMD (Standardized Mean Difference, 표준화 평균 차이)

AI 채점의 **편향(bias)을 잡는 지표**다. AI가 전반적으로 점수를 후하게 주거나 박하게 주는 경향을 수치화한다.

```
SMD = (AI 추천 점수 평균 - 기준 채점 점수 평균) / 표준편차
# |SMD| <= 0.20 을 목표로 설정
```

절댓값이 0.20을 넘으면 체계적 편향이 있다고 본다. 이 임계값은 교육 측정(educational measurement) 분야에서 표준적으로 통용된다.

### 4. 재현율 (Recall) — 저득점 구간 한정

일반적인 분류 지표지만, 여기서는 의미가 다르다. **저득점(하위 20%) 구간에서 AI가 해당 학생을 제대로 감지하는지** 확인하는 용도다. 고위험 판별력 검증이라고 보면 된다.

### 5. 열화 지수 (Degradation)

사람-사람 간 QWK 대비 사람-AI 간 QWK가 얼마나 떨어지는지 측정한다.

```
Degradation = 사람-사람 QWK - 사람-AI QWK <= 0.10
```

AI를 도입했을 때 품질이 얼마나 나빠지는지 보여주는 지표다. 이 수치를 명시하면 "AI가 사람보다 나쁜가?"라는 질문에 직접 답할 수 있다.

## 재현율 100%의 함정과 결정성 통제

LLM(Large Language Model, 대형 언어 모델)을 채점 엔진으로 쓸 때 흔히 저지르는 실수가 있다. 재현율 목표를 100%로 잡는 것이다.

**재현율 100%는 위험하다.** LLM은 같은 입력에 항상 같은 출력을 내지 않기 때문이다. 온도 파라미터(temperature)를 제어하지 않으면 동일한 글에 다른 점수를 줄 수 있다. 심사 현장에서 "어제는 3점이었는데 오늘은 4점이 나왔다"는 상황이 생기면 치명적이다.

현실적인 대응은 두 가지다:

1. **결정성 고정**: `temperature=0` + 결과 캐싱으로 동일 입력에 동일 출력 보장
2. **현실적 목표치 설정**: 재현율 95~98% 수준으로 잡고, 나머지는 사람 검토 플로우로 연결

문서에 "재현율 100% 지향"을 명시해놓으면, 검증 단계에서 반드시 실패한다. 달성 불가능한 기준은 방어력이 아니라 취약점이 된다.

## 마치며

지표를 고르는 일은 설계 결정이다. 숫자를 좋아 보이게 만드는 게 아니라, **무엇을 측정할 것인가**를 먼저 정해야 한다.

AES 시스템에 Accuracy 하나만 들고 나가면, "극단값에서 얼마나 틀리나요?", "편향은 없나요?", "사람 채점보다 얼마나 나쁜가요?" 이런 질문들에 답을 못 한다.

QWK, Pearson, SMD, 재현율, 열화 지수를 조합하면 **각각 다른 각도에서 시스템을 검증**한다. 어떤 지표도 혼자 완벽하지 않다. 그래서 여러 개를 쓴다.

목표치도 마찬가지다. 방어 가능한 수치는 문헌과 표준에 근거가 있어야 한다. "높게 잡으면 좋아 보이겠지"라는 생각으로 접근하면, 정작 검증 단계에서 무너진다.
