# 도입 (왜 이 글)

교육 콘텐츠 추천 시스템을 만들다 보면 어느 순간 이런 생각이 든다.  
"일단 키워드를 많이 달면 매칭이 잘 되지 않을까?"

나도 그렇게 생각했다. 그래서 콘텐츠마다 10개씩 keyword_tags를 달아두고 뿌듯해했다. 근데 막상 추천 결과를 보니 엉뚱한 콘텐츠끼리 묶이고, 정작 연결돼야 할 콘텐츠들은 서로 모르는 척했다.

3만여 건의 콘텐츠를 다시 태깅하면서 깨달은 건 단순했다.  
**키워드는 '표현'이 아니라 '연결 고리'다.**

---

## 키워드를 많이 달수록 추천이 망가지는 이유

keyword_tags(키워드 태그)는 콘텐츠를 설명하는 레이블이 아니다. 추천 시스템이 "이 콘텐츠와 저 콘텐츠는 같은 맥락이다"라고 판단하는 **공통 의미 단위**다.

키워드를 10개 달면 무슨 일이 생기냐면:

- `기초`, `실무`, `이해`, `전략` 같은 generic(범용적인) 단어가 끼어든다
- 그 generic 키워드 하나가 전혀 다른 도메인의 콘텐츠를 죄다 연결해버린다
- 결과적으로 추천 결과가 "전략 관련 콘텐츠 모음집"이 되어버린다

반대로 `2023년-4분기-영업팀-KPI-분석` 같은 지나치게 세부적인 키워드는 어떨까? 그 콘텐츠 하나에만 달려있으니 추천 신호로는 쓸모가 없다. 매칭 대상이 없는 외딴섬이 된다.

---

## 실제로 적용한 태깅 판단 기준

재태깅 작업을 하면서 정한 규칙은 이렇다.

**1. 판단 순서: 제목 → 설명 → 기존 태그**

```
입력: title + description + original keyword_tags
출력: 새 keyword_tags (2-3개)
```

제목만 보고 달지 않는다. 설명을 읽으면서 "이 콘텐츠가 실제로 다루는 것"이 무엇인지를 먼저 파악한다. 기존 태그는 참고하되, 너무 광범위하거나 너무 세부적이면 과감하게 버린다.

**2. 개수는 보통 2-3개**

```yaml
# 나쁜 예
keyword_tags: ["AI", "데이터", "자동화", "효율", "실무", "전략", "기초", "분석"]

# 좋은 예
keyword_tags: ["AI", "데이터", "자동화"]
```

숫자 자체가 목표가 아니다. 다른 콘텐츠와 공통점이 있는 키워드 2-3개가 10개짜리 잡동사니보다 훨씬 낫다.

**3. 이 단어들은 단독으로 달지 않는다**

```
generic 단독 금지 목록:
기초 / 실무 / 이해 / 전략 / 관리 / 분석
```

이 단어들은 콘텐츠 어디에나 붙을 수 있기 때문에 추천 신호로서의 변별력이 없다. "AI 기초"처럼 도메인과 묶이면 괜찮지만, `기초` 단독으로는 절대 달지 않는다.

## Cross-strand 매칭: 키워드가 연결고리가 되는 순간

cross-strand matching(교차 스트랜드 매칭)이란, 서로 다른 학습 경로(strand)에 있는 콘텐츠가 같은 키워드를 통해 연결되는 것이다.

예를 들어 `데이터 시각화`라는 키워드가 있다면, 이게 "Python 입문" 스트랜드의 콘텐츠에도 붙고, "비즈니스 분석" 스트랜드의 콘텐츠에도 붙을 수 있다. 그러면 추천 시스템은 두 콘텐츠를 자연스럽게 연결한다.

skill_strand(스킬 스트랜드, 콘텐츠의 학습 경로 분류)는 참고값이지 태깅의 기준이 아니다. 키워드는 strand 경계를 넘어서 **실제 의미 기반으로** 붙여야 한다.

```
cross-strand 매칭 허용 예시:
- "SQL 기초" 스트랜드  →  keyword_tags: ["SQL", "데이터베이스"]
- "데이터 분석" 스트랜드 →  keyword_tags: ["SQL", "데이터 분석"]
  → SQL이라는 공통 키워드로 두 콘텐츠가 자연스럽게 연결됨
```

이게 제대로 되면 "SQL을 공부하는 사람"에게 두 스트랜드의 콘텐츠가 함께 추천될 수 있다.

## 마치며

3만 건을 다시 태깅하면서 얻은 가장 큰 교훈은,  
**키워드 태깅은 콘텐츠를 '묘사'하는 작업이 아니라 콘텐츠 간 '관계'를 설계하는 작업이라는 것이다.**

많이 달수록 좋다는 생각을 버리고 나서야 추천 품질이 올라갔다.

보수적으로, 공통적으로, 의미 중심으로.  
이 세 가지만 기억하면 된다.
