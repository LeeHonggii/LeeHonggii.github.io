# 도입 (왜 이 글)

콘텐츠 메타데이터 키워드를 정리하다가 한 가지 착각을 오래 품고 있었다는 걸 깨달았다.
"이 콘텐츠를 잘 설명하는 키워드를 뽑으면 된다"는 생각이었다.

그런데 추천 시스템에서 키워드는 콘텐츠를 설명하는 게 아니라 **콘텐츠끼리 연결하는 매개체**다.
아무리 정확한 키워드라도 다른 콘텐츠와 공유되지 않으면 추천 매칭에서 아무 역할을 못 한다.
이 글은 그 전제를 바꾸고 나서 정제 기준이 어떻게 달라졌는지 기록한다.

---

## "좋은 키워드"의 기준이 잘못됐다

처음엔 제목과 설명을 읽고 콘텐츠를 가장 잘 표현하는 단어를 골랐다.
강의가 "딥러닝 모델 경량화"에 관한 거라면 `모델경량화`, `pruning`, `quantization` 같은 단어를 붙였다.

문제는 이런 단어들이 **이 강의에만 붙어 있다**는 것이다.
유사한 다른 강의가 `경량화기법`, `모델최적화`를 쓰고 있다면 두 콘텐츠는 키워드 기반 매칭에서 연결되지 않는다.
추천 시스템 입장에서는 서로 무관한 콘텐츠가 된다.

키워드를 보는 시야를 바꿔야 했다.

---

## 전체 키워드 풀을 먼저 봐야 하는 이유

실전에서 키워드를 검토할 때 지금은 세 가지를 함께 본다.

1. **제목 + 설명** — 콘텐츠가 무엇을 다루는지
2. **기존 `keyword_tags`** — 이미 붙어 있는 태그, 혹은 비어 있는 이유
3. **전체 키워드 풀** — 시스템 전체에서 실제로 사용 중인 태그 목록

세 번째가 핵심이다.
전체 풀을 보면 어떤 태그가 여러 콘텐츠에 걸쳐 있는지 보인다.
새로 태그를 붙일 때는 "이 콘텐츠를 정확히 표현하는가"가 아니라
**"이 태그를 가진 다른 콘텐츠와 연결되는가"**를 먼저 따진다.

배치 처리 시 입력 구조는 이렇게 만든다:

```json
{
  "batch": "AI_batch01",
  "rows": [
    {
      "cid": "content-001",
      "title": "딥러닝 기초 with PyTorch",
      "description": "신경망 구조를 코드로 구현하며 배우는 입문 강의",
      "keyword_tags": [],
      "pool_sample": ["딥러닝", "PyTorch", "신경망", "머신러닝", "AI기초"]
    }
  ]
}
```

`pool_sample`에 전체 풀의 일부를 넣어 모델(또는 작업자)이 기존 태그와 정합성을 유지한 채 판단하도록 한다.

---

## 2~3개가 실용적인 이유

태그 수에 대한 기준도 고민이 많았다.
많을수록 좋을 것 같지만, 태그가 많아지면 두 가지 문제가 생긴다.

첫째, **희소한 태그가 섞여 매칭 신호가 약해진다.**
10개 중 8개가 이 콘텐츠에만 있는 고유 태그라면 나머지 2개의 공통 태그 신호가 희석된다.

둘째, **관리 비용이 기하급수로 늘어난다.**
태그 수가 늘수록 중복, 오타, 의미 겹침을 추적하기 어려워진다.

반대로 1개만 있는 경우는 반드시 보강한다.
단일 태그는 연결 경로가 하나뿐이라 추천 다양성이 떨어진다.

결과물 형식은 단순하게 유지한다:

```json
{
  "batch": "AI_batch01",
  "results": [
    {
      "cid": "content-001",
      "keywords": ["딥러닝", "PyTorch"]
    }
  ]
}
```

그리고 작업이 끝난 후엔 항상 원본과 대조한다:

```
result.json ↔ metadata_v15.json mismatch = 0
```

mismatch가 0이 될 때까지 반영 전에 검수한다.

## Human-in-the-Loop — 자동화가 놓치는 것

**Human-in-the-Loop**(사람이 루프 안에 개입하는 방식)를 택한 이유가 있다.
LLM 기반 자동 태깅은 콘텐츠 단위로는 정확하다.
하지만 전체 풀에서 어떤 태그가 실제로 매칭에 쓰이는지, 어떤 태그가 이미 과포화인지는 **전체 분포를 보는 사람**이 더 잘 판단한다.

자동화는 초안을 빠르게 만드는 데 쓰고, 최종 판단은 풀 단위 맥락을 아는 사람이 한다.
이 둘의 역할 분담이 없으면 자동 태깅 결과를 그냥 추천 시스템에 꽂는 실수로 이어진다.

---

## 마치며

추천 시스템에서 키워드의 역할은 콘텐츠 설명이 아니라 연결이다.
잘 설명하는 태그가 아니라, **다른 콘텐츠와 만날 수 있는 태그**를 붙여야 한다.

그 기준은 단순하다:
- 전체 풀에 이미 있는 태그를 우선 쓴다
- 2~3개 범위를 지킨다
- 1개만 있으면 무조건 보강한다

작은 기준처럼 보이지만, 이 기준을 세우기 전과 후의 추천 매칭률은 체감상 확연히 달랐다.
```
