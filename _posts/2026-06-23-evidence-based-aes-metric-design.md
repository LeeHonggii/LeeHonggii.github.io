# 도입 (왜 이 글)

AI 자동채점 시스템을 만들어 제안서를 쓰다가 처음에는 이렇게 적었다.

> "정확도 85% 이상 달성"

그런데 심사위원이 "그게 정확히 무슨 정확도냐"고 물으면 뭐라 답할 것인가. 교육 평가 분야에서는 정확도(accuracy)는 점수가 딱 일치하는 비율일 뿐이고, ±1점 차이를 아예 오답으로 처리한다. 에세이 채점에서 5점 만점에 3점과 4점을 구분하지 못했다는 것이 0점짜리 오답과 같은 가중치가 된다는 뜻이다. 이 글은 그 문제를 정면으로 파고들어, 실제 심사에서 방어 가능한 지표 체계를 어떻게 설계했는지 정리한다.

---

## 왜 정확도 하나로는 부족한가

분류 문제에서 정확도는 직관적이다. 하지만 자동채점은 **순서가 있는 정수 레이블**(1점, 2점, 3점…)을 다루는 회귀에 가깝다. 3점을 2점으로 틀리는 것과 3점을 0점으로 틀리는 것은 심각도가 전혀 다르다. 정확도는 이 두 케이스를 동등하게 취급한다.

또한 정확도는 **채점자 편향**을 잡지 못한다. AI가 전반적으로 점수를 1점씩 높게 주는 경향이 있어도, 우연히 일치율이 높으면 "정확도 높음"으로 나올 수 있다. 실제 교육 현장에서는 이게 큰 문제다.

---

## 핵심 지표 4개 체계

### 1. QWK — 인접 오류를 용인하는 일치 지표

**QWK(Quadratic Weighted Kappa)**: 채점자 간 일치도를 측정하되, 점수 차이가 클수록 페널티를 제곱으로 부과하는 지표. ASAP(Automated Student Assessment Prize) 같은 국제 대회 표준이다.

```
QWK >= 0.70  → 허용 기준 (충분한 일치)
QWK >= 0.80  → 우수
```

단순 정확도보다 "얼마나 가깝게 틀렸나"를 반영하므로, 에세이처럼 주관성이 있는 채점에 적합하다.

### 2. Pearson 상관계수 — 분포 수준의 선형 관계

**Pearson r**: AI 점수와 인간 채점 점수 사이의 선형 상관. Spearman도 자주 쓰이지만, 자동채점 표준 문헌(e.g., ETS 내부 보고서, ASAP 논문)에서는 Pearson을 주로 보고한다. 심사에서 근거를 물으면 Pearson이 훨씬 답하기 쉽다.

```
Pearson >= 0.70  → 기준치
```

### 3. SMD — 편향 탐지

**SMD(Standardized Mean Difference)**: AI가 체계적으로 점수를 높이거나 낮추는지를 표준화된 단위로 본다.

```
SMD = (AI 평균 점수 - 기준 채점 평균 점수) / 표준편차
```

절댓값이 0.2 미만이면 편향이 작은 것으로 본다. 이 숫자 하나만 있어도 "AI가 관대하게 채점한다/엄격하게 채점한다"는 주장을 수치로 막을 수 있다.

### 4. Degradation — 인간 대비 열화율

**Degradation(열화)**: 사람-사람 간 QWK 대비, 사람-AI 간 QWK가 얼마나 떨어지는지.

```
Degradation = 사람-사람 QWK - 사람-AI QWK <= 0.10
```

예를 들어 동일 답안에 대해 두 사람 채점자가 QWK 0.82로 일치한다면, AI는 최소 0.72 이상을 목표로 잡는 것이다. 절대 수치보다 **상대적 하락폭**으로 기준을 잡으면 데이터셋마다 기준이 달라지는 문제도 해결된다.

## LLM 기반 채점에서 재현율 100% 목표는 왜 위험한가

LLM 채점을 도입하면 재현율(recall) 100%가 당연히 달성되는 것처럼 보인다. 모델이 모든 입력에 숫자를 출력하니까. 하지만 여기서 위험이 생긴다.

- **비결정성**: temperature > 0이면 같은 답안에 다른 점수가 나올 수 있다. 재현율 100%처럼 보여도 일관성이 없다.
- **허상 수치**: 모든 케이스에 점수를 출력하지만 그게 의미 있는 점수냐는 별개다.

현실적인 대응:

```python
# 결정성 통제 예시
response = client.messages.create(
    model="claude-sonnet-4-6",
    temperature=0,       # 동일 입력 → 동일 출력 보장
    ...
)
# + 입력-출력 쌍 캐싱으로 동일 답안 재채점 방지
```

재현율 목표는 100%보다 "채점 실패율 1% 미만" 같은 형식으로 바꾸는 게 방어 가능하다.

## 마치며

지표를 고를 때 "익숙하니까"가 아니라 "표준 문헌이 쓰니까"를 기준으로 삼게 됐다. Spearman 대신 Pearson을 쓴 이유, 정확도 대신 QWK를 쓴 이유를 한 문장씩 설명할 수 있어야 심사를 통과한다.

요약하면 이렇다:

| 지표 | 측정하는 것 | 기준값 |
|---|---|---|
| QWK | 인접 오류 허용 일치도 | ≥ 0.70 |
| Pearson r | 분포 수준 선형 상관 | ≥ 0.70 |
| SMD | 채점 편향 크기 | \|SMD\| < 0.20 |
| Degradation | 인간 대비 성능 하락 | ≤ 0.10 |

이 네 개를 함께 들고 가면, "AI 채점 믿어도 되냐"는 질문에 데이터로 답할 수 있다.
